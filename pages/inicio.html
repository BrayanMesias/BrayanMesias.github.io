Realizar la implementación de un Modelo Relacional sobre un SGBD permite al usuario como materializar, diseñar y conocer la importancia que tiene un diagrama entidad-relación hoy en día y que a partir de ahí y en función del uso que se le vaya a dar a la base de datos se escogerá la representación conceptual más adecuada para la misma. Todo ello para conseguir finalmente una representación física de nuestro proyecto a través del lenguaje SQL.





Para iniciar con la implementación de un modelo Relacional se debe tener en cuenta 3 funciones básicas que debe realizar un SGBD (Sistema Gestor de Base de Datos):

  • Función de descripción o definición.
  • Función de manipulación.
  • Función de control.
